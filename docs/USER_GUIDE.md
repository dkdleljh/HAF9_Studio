# HAF-9_Studio 사용자 가이드

HAF-9_Studio는 프로그래밍 가능한 공간 운영 체제입니다. 이 가이드는 운영자가 시스템을 효과적으로 사용하는 방법을 설명합니다.

## 시스템 개요

HAF-9_Studio는 다음을 제공합니다:

1. **공간 디지털 트윈**: 2D 그리드 기반 건물 모델
2. **의도 처리**: 자연어를 구조화된 명령으로 변환
3. **패치 제안**: 공간 수정을 위한 후보 자동 생성
4. **안전 검증**: 패치 적용 전 안전성 확인
5. **포렌식 추적**: 모든 작업의 완전한 감사 로그

## 시작하기

### 1. 시나리오 로드

시스템을 시작하면 기본 예시 시나리오가 자동으로 로드됩니다:
- 20x20 그리드
- 3개 존 (Operations, Assembly, Laboratory)
- 2개 출구
- 여러 장애물

화면 왼쪽의 "시나리오Reload" 버튼을 클릭하여 언제든지 재로드할 수 있습니다.

### 2. 공간 디지털 트윈 확인

중앙 패널에 2D 그리드가 표시됩니다:
- 파란색: Zone 1 (Operations)
- 보라색: Zone 2 (Assembly)
- 주황색: Zone 3 (Laboratory)
- 녹색: 출구
- 빨간색: 장애물

오른쪽 패널에는 각 존의 현재 메트릭이 표시됩니다:
- 대피 시간 (초)
- 인구 밀도 (%)
- 피크 스트레스
- 접근성 (%)
- 소음 (dB)

## 인텐트 입력

### 자연어 입력

상단 "인텐트 입력" 패널에 원하는 변경 사항을 입력하세요.

**예시 인텐트:**
```
Reduce evacuation time in zone3 to under 12 seconds while maintaining structural safety
```

또는:
```
Improve accessibility in zone1
```

### 인텍스트 처리 흐름

인텐트를 입력하면 시스템은 다음 단계를 자동으로 수행합니다:

1. **파싱**: 자연어를 구조화된 인텐트로 변환
2. **DSL 생성**: 인텐트 DSL 생성
3. **IR 컴파일**: 제약 IR로 변환
4. **패치 생성**: 여러 후보 패치 생성

각 단계는 화면 상단의 상태 배지에 표시됩니다.

### 파싱 결과 확인

인턴트가 파싱되면 "파싱된 인턴트" 패널에 다음 정보가 표시됩니다:
- 인턴트 ID
- 유형 (evacuation_optimization, safety_enhancement 등)
- 대상 존
- 목적
- 모호성 점수 및 수준
- 경고 (있는 경우)

## 패치 검토 및 적용

### 패치 목록

생성된 패치 후보는 "패치 후보" 패널에 표시됩니다.
각 패치는 다음 정보를 포함합니다:
- 패치 유형
- 설명
- 대상 셀 수
- 예상 효과
- 우선순위

### 패치 선택

패치를 클릭하면 선택됩니다. 선택된 패치는 안전성 검증 대상이 됩니다.

### 안전성 검증

패치를 선택하면 자동으로 안전성 검증이 실행됩니다.

**결정 유형:**
- **APPROVED (승인됨)**: 패치를 적용할 수 있습니다
- **REJECTED (거부됨)**: 안전성 문제가 있어 적용할 수 없습니다
- **REQUIRES_HUMAN_REVIEW (인계 검토 필요)**: 수동 검토가 필요합니다

### 패치 적용

"패치 적용" 버튼을 클릭하여 선택된 패치를 적용합니다.

**주의:** 패치를 적용하면 시뮬레이션 상태가 변경됩니다. 이 작업은 포렌식 원장에 기록됩니다.

## 메트릭 비교

패치를 적용하면 메트릭 패널이 업데이트되어 변경 사항을 반영합니다.

## 포렌식 원장

오른쪽 하단 패널에는 모든 작업의 로그가 표시됩니다.

**기록되는 이벤트:**
- 인텐트 수신
- 인텐트 파싱
- DSL 생성
- IR 컴파일
- 패치 생성
- 안전성 결정
- 패치 적용/거부
- 메트릭 변경

각 이벤트를 클릭하면 상세 정보를 볼 수 있습니다.

## 리플레이

포렌식 원장을 사용하여 시스템 상태 변경을 리플레이할 수 있습니다.

`/api/replay/timeline` 엔드포인트를 사용하여 타임라인을 조회하고,
`/api/replay/replay-from/{event_id}`로 특정 이벤트부터 리플레이할 수 있습니다.

## 문제 해결

### 시나리오가 로드되지 않는 경우

백엔드가 실행 중인지 확인:
```bash
curl http://localhost:8000/api/health
```

### 인텍스트 처리 오류

오류가 발생하면 화면에 오류 메시지가 표시됩니다. 자세한 내용은 포렌식 원장을 확인하세요.

### 패치가 거부된 경우

"안전성 검증 결과" 패널에 거부 이유가 표시됩니다. 문제를 해결하려면 다른 패치를 시도하거나 인텍스트를 수정하세요.

## 팁

1. **명확한 인텍스트 사용**: 구체적인 수치(예: "12초 이하")를 포함하면 모호성이 줄어듭니다

2. **메트릭 확인**: 패치 적용 전후의 메트릭을 비교하여 효과를 확인하세요

3. **원장 활용**:出了问题時 원장을 검토하여发生了什么を正確に把握できます

4. **패치 순위**: 우선순위가 높은 패치부터 시도하세요
